How Piko Works

Piko creates isolated development environments by combining three tools you already know: Git worktrees for code isolation, tmux for terminal sessions, and optionally Docker for container orchestration.

Each environment gets its own branch, its own directory, its own containers, and its own ports. No conflicts between environments.

Getting Started

go install github.com/bearsignals/piko@latest

cd ~/projects/myapp
piko init
piko create feature-auth

That's it. You're now in a tmux session with your worktree, containers running, and everything isolated. When you're done: 

piko destroy feature-auth

The Primitives

Piko provides four primitives. You compose them however you need.

Worktrees — Each environment is a Git worktree at .piko/worktrees/<name>. Your code is isolated per branch.

Tmux sessions — Each environment gets a session named piko/<project>/<name> with windows for shell, services, and logs.

Environment variables — Piko sets PIKO_* variables in your session. Use them to derive ports, configure databases, or anything else.

Data directories — Each environment gets .piko/data/<name> for databases, caches, build outputs—anything that should be isolated.

Environment Variables

These are available in your tmux session and scripts:

PIKO_ENV_NAME   # "feature-auth"
PIKO_ENV_ID     # unique integer (useful for ports)
PIKO_ENV_PATH   # path to worktree
PIKO_DATA_DIR   # path to data directory
PIKO_ROOT       # project root
PIKO_DB_PORT    # allocated port (Docker mode)

Derive ports from PIKO_ENV_ID to avoid conflicts: 

# Shell
npm run dev -- --port $((3000 + PIKO_ENV_ID))

# vite.config.js
server: { port: 3000 + parseInt(process.env.PIKO_ENV_ID || 0) }

Use PIKO_DATA_DIR for local databases:

DATABASE_URL=sqlite:${PIKO_DATA_DIR}/dev.db

Configuration

Piko works without configuration. If you need to customize, create .piko.yml in your project root: 

scripts:
  setup: |
    ln -s "$PIKO_ROOT/.env.local" .env.local
    npm install

  run: |
    DATABASE_URL="postgres://localhost:$PIKO_DB_PORT/dev" \
    npm run dev

setup runs after environment creation. run is executed by piko env run <name>.

Docker Mode vs Simple Mode

If your project has docker-compose.yml, piko orchestrates containers with isolated networks and dynamically allocated ports.

Without Docker Compose, piko runs in simple mode: worktrees + tmux + data directories. You handle isolation using the environment variables.

Mode is auto-detected. No configuration needed.

Navigation

piko attach feature-auth   # attach to session
piko env pick              # fuzzy-pick with fzf
piko env ls                # list all environments

Inside tmux, standard navigation works: Ctrl-b s for session picker, Ctrl-b w for windows.

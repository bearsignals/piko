piko GitHub

Isolated dev environments
from git worktrees

Work on multiple features simultaneously. Each worktree gets its own containers, ports, and tmux session. Switch contexts instantly.

go install github.com/gwuah/piko@latest copy
$ piko init
Initialized piko for myapp

$ piko create feature-auth
Created worktree at .piko/worktrees/feature-auth
Started containers (piko-myapp-feature-auth)
Created tmux session

$ piko create feature-payments
Created worktree at .piko/worktrees/feature-payments
Started containers (piko-myapp-feature-payments)
Created tmux session

$ piko list
NAME STATUS BRANCH
feature-auth running feature-auth
feature-payments running feature-payments

Why piko?

Instant context switching

Jump between features with piko attach. Each environment is a tmux session with pre-configured windows.

Zero port conflicts

Each environment gets unique ports derived from its ID. No more "address already in use" errors.

Isolated data

Separate databases, caches, and build artifacts per environment. Test migrations safely.

Works with your stack

Uses your existing docker-compose.yml. Or skip Docker entirely with simple mode.

Two modes, same workflow

D

Docker mode

For projects with docker-compose.yml. Each environment gets isolated containers with unique ports.

worktree + containers + tmux
S

Simple mode

For projects without Docker. Get isolated worktrees, data directories, and env vars for port derivation.

worktree + data dir + tmux

Get started

1. Install piko

go install github.com/gwuah/piko@latest

2. Initialize your project

cd your-project && piko init

3. Create an environment

piko create my-feature

Requires git, tmux. Docker optional.

piko

Isolated development environments using Git worktrees, tmux, and Docker.

"I want to work on feature-auth and feature-payments simultaneously,
each with their own database state, without conflicts."

Philosophy

Primitives over prescriptions. Provides building blocks, lets you compose.
Composable. Leverages Git, tmux, Docker. No lock-in.
Zero config by default. Just works, configure only if needed.
Graceful degradation. Works with or without Docker.

Workflow

$ piko init # initialize project
$ piko create feature-auth # worktree + containers + tmux
$ piko attach feature-auth # jump into the environment
$ piko destroy feature-auth # tear it all down

What it provides

worktrees + tmux + env vars + data dirs

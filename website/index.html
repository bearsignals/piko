piko

Work on multiple features at once.

Isolated dev environments with their own branches, containers, and ports.

$ go install github.com/bearsignals/piko@latest

Usage

$ piko init
$ piko create feature-auth
$ piko attach feature-auth

What you get

Isolated branches

Git worktrees at .piko/worktrees/<name>

Isolated containers

Each environment gets its own Docker network and ports

Persistent sessions

tmux sessions you can attach/detach from

Isolated data

Separate directories for databases and caches

How it works

piko create sets up a Git worktree, starts Docker containers on unique ports, and opens a tmux session. piko attach reconnects you. piko destroy tears everything down. Works with or without Docker.

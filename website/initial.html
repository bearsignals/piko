piko Source

Isolated dev environments
from git worktrees.

Work on multiple features simultaneously. Each worktree gets its own containers, ports, and tmux session.

$ go install github.com/bearsignals/piko@latest Copy
~ piko init Initialized piko configuration
~ piko create feature-auth Created worktree at .piko/worktrees/feature-auth Started containers (piko-myapp-feature-auth) Attached to tmux session
~ piko list NAME STATUS PORTS feature-auth running 8081:8080 feature-payments running 8082:8080
Capabilities

Zero Port Conflicts

Piko mathematically derives unique ports for each environment based on the feature name. No more "Address already in use".

Context Isolation

Jump between features with piko attach. Every environment is a self-contained tmux session with its own state.

Data Sovereignty

Database migrations in one branch won't break the others. Data directories are isolated per environment.

Stack Agnostic

Uses your existing docker-compose.yml. If you don't use Docker, Simple Mode isolates binaries and env vars.

Modes

Docker Mode

For containerized stacks. Piko handles port mapping and container naming automatically.

Simple Mode

For binary/script stacks. Piko manages environment variables and data directories.
